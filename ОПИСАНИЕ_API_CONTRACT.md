# Описание модуля films-api-contract

Модуль **films-api-contract** — библиотека контракта REST API системы управления каталогом фильмов. Содержит интерфейсы эндпоинтов, DTO, правила валидации, описание OpenAPI и GraphQL-схему. Сервис **rest-service** подключает этот модуль как зависимость и реализует интерфейсы в контроллерах.

---

## FilmApi

**FilmApi** — Java-интерфейс с аннотациями Spring MVC, который описывает REST-контракт для работы с фильмами. Контроллер `FilmController` в сервисе `rest-service` реализует данный интерфейс и не дублирует объявление URL и HTTP-методов.

Интерфейс предназначен для выполнения CRUD-операций над сущностью «фильм»: получение списка и отдельной записи, создание, полное и частичное обновление, удаление. Все методы интерфейса приведены в таблице 1.

**Таблица 1 – методы FilmApi**

| HTTP-метод | Эндпоинт | Действие |
|------------|----------|----------|
| GET | `/api/films` | Возвращает постраничный список фильмов с фильтрацией |
| GET | `/api/films/{id}` | Получение фильма по идентификатору |
| POST | `/api/films` | Создание нового фильма |
| PUT | `/api/films/{id}` | Полное обновление фильма. Заменяет все поля ресурса |
| PATCH | `/api/films/{id}` | Частичное обновление. Изменяются только переданные поля |
| DELETE | `/api/films/{id}` | Удаление фильма |

Ответы интерфейса возвращаются вместе с **гиперссылками HATEOAS** (`EntityModel`, `PagedModel`). Клиенту не нужно заранее знать URL связанных ресурсов — сервер включает их в блок `_links` (например, ссылка на режиссёра фильма или на коллекцию всех фильмов).

HTTP-методы разделены по семантике:

- **POST** — создание нового ресурса;
- **PUT** — полная замена существующего ресурса;
- **PATCH** — точечное изменение отдельных полей;
- **DELETE** — удаление ресурса.

Аннотации **@Operation** и **@ApiResponse** используются для автоматической генерации интерактивной документации Swagger и описания возможных кодов ответа (200, 201, 204, 400, 404, 409).

Аннотация **@Valid** на теле запроса включает автоматическую проверку входных данных до передачи в сервисный слой.

Метод получения списка фильмов поддерживает **пагинацию** (параметры `page`, `size`) и **фильтрацию** по идентификатору режиссёра, жанру, году выхода и подстроке в названии. Фильтры можно комбинировать в одном запросе.

**Бизнес-логика, реализуемая через FilmApi:**

- просмотр каталога фильмов с постраничной навигацией;
- поиск и фильтрация записей;
- добавление нового фильма с привязкой к режиссёру;
- редактирование данных фильма (полное или выборочное);
- удаление фильма из каталога.

**Возможные ошибки:**

- **404** — фильм или режиссёр с указанным идентификатором не найден;
- **400** — ошибка валидации входных данных (некорректный IMDb ID, пустое название и т.д.);
- **409** — фильм с таким IMDb ID уже существует в каталоге.

## DirectorApi

**DirectorApi** — Java-интерфейс с аннотациями Spring MVC, описывающий REST-контракт для управления режиссёрами. Реализуется классом `DirectorController` в модуле `rest-service`.

Интерфейс обеспечивает CRUD-операции над сущностью «режиссёр», а также доступ к связанным фильмам через суб-ресурс. Все методы представлены в таблице 2.

**Таблица 2 – методы DirectorApi**

| HTTP-метод | Эндпоинт | Действие |
|------------|----------|----------|
| GET | `/api/directors` | Возвращает постраничный список режиссёров |
| GET | `/api/directors/{id}` | Получение режиссёра по идентификатору |
| POST | `/api/directors` | Создание нового режиссёра |
| PUT | `/api/directors/{id}` | Полное обновление режиссёра |
| PATCH | `/api/directors/{id}` | Частичное обновление режиссёра |
| DELETE | `/api/directors/{id}` | Удаление режиссёра и всех его фильмов (каскадно) |
| GET | `/api/directors/{id}/films` | Получение списка фильмов данного режиссёра |

Как и в FilmApi, ответы содержат **HATEOAS-ссылки**, что позволяет клиенту навигировать по API без жёстко заданных URL.

Семантика HTTP-методов аналогична FilmApi: POST для создания, PUT для полной замены, PATCH для частичного изменения, DELETE для удаления.

**Swagger-аннотации:**

- **@Tag** — группировка эндпоинтов в раздел «Directors» документации;
- **@Operation** — текстовое описание назначения метода;
- **@ApiResponse** — перечень HTTP-кодов и их смысл;
- **@Parameter** — описание query- и path-параметров.

**Валидация:**

- **@Valid** — запуск Bean Validation для тела запроса;
- **@Valid @RequestBody** — проверка DTO до вызова бизнес-логики.

Список режиссёров возвращается с **пагинацией**. Эндпоинт `/api/directors/{id}/films` реализует REST-паттерн **суб-ресурса** и эквивалентен фильтрации `GET /api/films?directorId={id}`, но явнее отражает иерархию «режиссёр → фильмы».

**Бизнес-логика, реализуемая через DirectorApi:**

- просмотр списка режиссёров;
- получение карточки режиссёра с биографией и счётчиком фильмов;
- регистрация нового режиссёра в каталоге;
- изменение данных режиссёра;
- удаление режиссёра вместе с его фильмами;
- просмотр фильмографии режиссёра.

**Возможные ошибки:**

- **404** — режиссёр с указанным ID не найден;
- **400** — ошибка валидации (некорректный email, пустое имя и т.д.).

## DTO

### FilmRequest

Класс предназначен для **передачи данных от клиента к серверу** при создании фильма (`POST /api/films`). Реализован как **record** — неизменяемый тип данных с автоматически сгенерированными конструктором, геттерами и методами `equals()`, `hashCode()`, `toString()`.

Record выбран потому, что DTO запроса не наследует другие классы и служит исключительно для транспортировки данных. Принцип **разделения ответственности**: класс не содержит бизнес-логики, все проверки выполняются до попадания данных в сервисный слой.

Используется кастомная аннотация **@ValidImdbId** — проверяет формат IMDb ID (`tt` + 7–8 цифр).

**Поля класса:**

- **title** — название фильма. Обязательное, не более 500 символов;
- **imdbId** — уникальный идентификатор фильма в базе IMDb. Обязательное, формат `tt1375666`;
- **directorId** — идентификатор режиссёра. Обязательное;
- **description** — текстовое описание. Необязательное, до 5000 символов;
- **genre** — жанр. Необязательное, до 100 символов;
- **publishedYear** — год выхода фильма. Необязательное, от 1 до 9999;
- **language** — код языка (ISO 639-1). Необязательное, 2–5 символов.

---

### FilmResponse

**FilmResponse** — DTO для **ответа сервера клиенту** с информацией о фильме. В отличие от record-запросов, реализован как **класс**, расширяющий `RepresentationModel<FilmResponse>`, поскольку record в Java не может наследовать другие классы, а для HATEOAS требуется базовый класс со ссылками.

**Поля:** id, title, imdbId, director (вложенный `DirectorResponse`), description, genre, publishedYear, language, createdAt, updatedAt.

**Аннотации:**

- **@Builder** — удобное построение объекта через Lombok;
- **@Relation(collectionRelation = "films", itemRelation = "film")** — имена коллекции и элемента в JSON при сериализации HATEOAS;
- **@JsonInclude(Include.NON_NULL)** — поля со значением null не включаются в JSON;
- **@EqualsAndHashCode(callSuper = false)** — HATEOAS-ссылки не участвуют в сравнении объектов.

---

### UpdateFilmRequest

DTO для **полного обновления** фильма (`PUT /api/films/{id}`). Заменяет все редактируемые поля ресурса. Режиссёр (**directorId**) изменить нельзя — для смены режиссёра создаётся новая запись фильма.

**Поля:** title, imdbId, description, genre, publishedYear, language — те же ограничения валидации, что и в FilmRequest, за исключением directorId.

---

### PatchFilmRequest

**PatchFilmRequest** — DTO для **частичного обновления** фильма (`PATCH /api/films/{id}`). Все поля необязательны: в теле запроса передаются только изменяемые атрибуты, остальные остаются без изменений. Семантика соответствует JSON Merge Patch (RFC 7396).

---

### DirectorRequest

DTO для **создания и полного обновления** режиссёра (`POST` и `PUT`). Реализован как **record**.

**Поля:**

- **firstName** — имя. Обязательное, до 100 символов;
- **lastName** — фамилия. Обязательное, до 100 символов;
- **email** — адрес электронной почты. Необязательное, проверяется аннотацией **@Email**;
- **bio** — биография. Необязательное, до 2000 символов;
- **birthDate** — дата рождения. Необязательное, должна быть в прошлом (**@Past**);
- **nationality** — национальность. Необязательное, до 100 символов.

Класс отвечает только за транспортировку и валидацию данных на границе API.

---

### DirectorResponse

**DirectorResponse** — DTO ответа с данными режиссёра. Расширяет `RepresentationModel<DirectorResponse>` для поддержки HATEOAS.

**Поля:** id, firstName, lastName, fullName (вычисляемое полное имя), email, bio, birthDate, nationality, filmsCount (количество фильмов в каталоге).

**Аннотации:** @Builder, @Relation(collectionRelation = "directors", itemRelation = "director"), @JsonInclude(NON_NULL), @EqualsAndHashCode(callSuper = false).

---

### PatchDirectorRequest

**PatchDirectorRequest** — DTO для частичного обновления режиссёра (`PATCH /api/directors/{id}`). Все поля опциональны; непереданные атрибуты сохраняют прежние значения.

---

### ErrorResponse

**ErrorResponse** — стандартный DTO ответа об ошибке в формате **RFC 7807 Problem Details**.

**Содержит:**

- **status** — HTTP-код (404, 400, 409 и т.д.);
- **type** — URI типа ошибки для программной обработки;
- **title** — краткое название ошибки;
- **detail** — подробное описание конкретного случая;
- **instance** — URI запроса, вызвавшего ошибку;
- **timestamp** — момент возникновения ошибки (UTC);
- **fieldErrors** — список ошибок по полям (для ответа 400 при валидации).

Вложенный record **FieldError** описывает одну ошибку поля: имя поля, отклонённое значение и текст сообщения.

---

### PagedResponse

**PagedResponse&lt;T&gt;** — универсальный record для **стандартизации постраничных ответов** API.

**Содержит:**

- **content** — элементы текущей страницы;
- **pageNumber** — номер страницы (с нуля);
- **pageSize** — размер страницы;
- **totalElements** — общее число записей;
- **totalPages** — общее число страниц;
- **last** — признак последней страницы.

---

## FilmsApiContractConfig

**FilmsApiContractConfig** — конфигурационный класс централизованной **OpenAPI-документации** API. Задаёт глобальные параметры Swagger, на которые ссылаются интерфейсы FilmApi и DirectorApi.

**Таблица 3 – основные элементы FilmsApiContractConfig**

| Элемент | Описание |
|---------|----------|
| @OpenAPIDefinition | Общая информация об API: название «Films API», версия, описание, адрес сервера |
| @SecurityScheme | Схема авторизации Bearer JWT |
| SECURITY_SCHEME_BEARER | Константа `"bearerAuth"`, используется в @SecurityRequirement на методах API |

Все защищённые методы FilmApi и DirectorApi ссылаются на единую схему безопасности, что обеспечивает согласованное описание авторизации во всей документации Swagger UI.

## Exception

### ResourceNotFoundException

**ResourceNotFoundException** — пользовательское **непроверяемое исключение** (наследует `RuntimeException`). Выбрасывается, когда запрашиваемый ресурс отсутствует в хранилище.

**Когда возникает:** при обращении к несуществующему фильму или режиссёру (GET, PUT, PATCH, DELETE по неверному id; создание фильма с несуществующим directorId).

**Пример сообщения:** `Film with id=42 not found`.

Обрабатывается в `GlobalExceptionHandler` сервиса rest-service и преобразуется в HTTP-ответ **404 Not Found**.

---

### ImdbIdAlreadyExistsException

**ImdbIdAlreadyExistsException** — непроверяемое исключение, сигнализирующее о **нарушении уникальности IMDb ID**.

**Когда возникает:** при создании или обновлении фильма, если другая запись каталога уже использует указанный imdbId.

**Пример сообщения:** `Film with IMDb ID=tt1375666 already exists`.

Преобразуется в HTTP-ответ **409 Conflict**.

---

## Validation

### ValidImdbId

**ValidImdbId** — **кастомная аннотация валидации**, проверяющая корректность формата IMDb ID. Применяется к полям imdbId в FilmRequest, UpdateFilmRequest и PatchFilmRequest.

Допустимый формат: строка вида `tt` + 7 или 8 цифр (например, `tt1375666`, `tt0468569`). Пустое значение считается допустимым на уровне валидатора — за обязательность отвечает отдельная аннотация **@NotBlank**.

**Параметры аннотации:**

- **message()** — текст ошибки по умолчанию;
- **validatedBy** — класс `ImdbIdValidator`, реализующий логику проверки.

---

### ImdbIdValidator

**ImdbIdValidator** — класс-валидатор, реализующий интерфейс `ConstraintValidator<ValidImdbId, String>`.

Проверяет строку регулярным выражением `^tt\d{7,8}$` (без учёта регистра). Если значение null или пустая строка — валидация пропускается (ответственность @NotBlank). При несоответствии формату возвращается false, и клиент получает ошибку 400 с описанием в ErrorResponse.
